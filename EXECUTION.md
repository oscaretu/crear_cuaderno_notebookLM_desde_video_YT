# NotebookLM API - Execution Guide

## Project Structure

```
/backend/          # FastAPI - REST API
/frontend/         # React + Vite - Web Interface
```

---

## Backend (FastAPI)

### Prerequisites

```bash
cd backend
pip install -r requirements.txt
playwright install chromium
```

> **IMPORTANT**: Keep `yt-dlp` updated. YouTube frequently changes its API:
> ```bash
> cd backend
> source venv/bin/activate
> pip install --upgrade yt-dlp
> ```

### Run the Server

```bash
cd backend
source venv/bin/activate
python -m app.main --host 127.0.0.1 --port 8000
```

Server available at: **http://127.0.0.1:8000**

### Parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| `--host` | Host to bind to | `127.0.0.1` |
| `--port` | Port | `8000` |
| `--debug` | Enable debug logging | `false` |

### Enable Debug

You have three options:

```bash
# 1. CLI argument
python -m app.main --debug

# 2. Environment variable
DEBUG=true python -m app.main

# 3. .env file
echo "DEBUG=true" > backend/.env
```

### API Documentation

- Swagger UI: http://127.0.0.1:8000/docs
- ReDoc: http://127.0.0.1:8000/redoc

### Available Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/health` | Health check |
| GET | `/api/auth/status` | Verify authentication |
| GET | `/api/auth/profiles?username=xxx` | List Firefox profiles |
| POST | `/api/auth/extract-cookies` | Extract Firefox cookies |
| POST | `/api/notebooks` | Create notebook from YouTube |
| GET | `/api/notebooks` | List notebooks |
| GET | `/api/notebooks/{id}` | View details and artifacts |
| POST | `/api/notebooks/{id}/artifacts` | Generate artifacts |

## Web Interface Features

### Bookmarklet - Create Notebook from YouTube

To quickly create a notebook from a YouTube video, use the URL with hash parameter:

```
http://127.0.0.1:3000/#/create?url=https://www.youtube.com/watch?v=VIDEO_ID
```

**Bookmarklet for browser:**
```javascript
javascript:(function(){window.open('http://127.0.0.1:3000/#/create?url='+encodeURIComponent(window.location.href),'_blank');})()
```

### Default Firefox Profile

1. Go to the "Authentication" tab
2. Select a profile from the dropdown
3. Enable "Use this profile as default"
4. The profile will be saved in the browser and used automatically

To delete the default profile, click "✕ Clear default profile".

The current profile is shown in the application header.

### Notebook Summary

When viewing notebook details, the summary generated by NotebookLM when sources were added is automatically displayed.

### Copy and Download Reports

In the artifacts section, reports show three options:

- **View**: Opens content in a modal with markdown rendering
- **Copy**: Copies content to clipboard
- **Download**: Saves the report as a `.md` file with:
  - Blockquote header indicating the notebook title and URL
  - Filename: `Notebook Title -- Report Title -- YYYYMMDD_HHMM.md`

Example of saved header:
```
> This document is a report of the notebook **Notebook Title**
> [https://notebooklm.google.com/notebook/...](https://notebooklm.google.com/notebook/...)
```

### Notebook Sorting

By default, notebooks are sorted by creation date (newest first).

## API Usage Examples

### 1. Verify Authentication Status

```bash
curl http://127.0.0.1:8000/api/auth/status
```

### 2. Extract Firefox Cookies

```bash
curl -X POST http://127.0.0.1:8000/api/auth/extract-cookies \
  -H "Content-Type: application/json" \
  -d '{"username": "oscar", "profile": "default-release"}'
```

### 3. Create Notebook from YouTube

```bash
curl -X POST http://127.0.0.1:8000/api/notebooks \
  -H "Content-Type: application/json" \
  -d '{
    "youtube_url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ",
    "language": "es",
    "artifacts": ["report", "mind_map", "data_table"]
  }'
```

### 4. List Notebooks

```bash
curl http://127.0.0.1:8000/api/notebooks
```

### 5. Generate Artifacts

```bash
curl -X POST http://127.0.0.1:8000/api/notebooks/{notebook_id}/artifacts \
  -H "Content-Type: application/json" \
  -d '{
    "artifact_types": ["audio", "slides"],
    "language": "es"
  }'
```

## Notes

- The backend requires Google authentication to work with NotebookLM
- Use the original `extraer_cookies_firefox.py` script or the `/api/auth/extract-cookies` endpoint to obtain credentials
- Cookies are stored in `~/.notebooklm/storage_state.json`
